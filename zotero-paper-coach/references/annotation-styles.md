# Annotation styles

Use this file to map user intent to annotation behavior.

## Styles

### mentor
Best for beginners and guided reading.

Behavior:
- prefer section-anchor highlights over many micro-highlights
- add plain-language notes generously
- explain why a passage matters
- include reading order nudges

### top-student
Best for exam prep and efficient study.

Behavior:
- mark high-yield passages only
- prefer concise notes
- emphasize definitions, result statements, and compare/contrast ideas
- add `[复述]` notes for memorable takeaways

### critical-researcher
Best for literature review and advanced reading.

Behavior:
- emphasize assumptions, limitations, evaluation weaknesses, baseline choices, and overclaim risk
- highlight gap statements and novelty claims
- add more pink and orange annotations than mentor mode

### executive-brief
Best for busy readers.

Behavior:
- annotate sparingly
- focus on abstract, contribution, strongest results, and one major limitation
- prefer short notes or none

## Color schemes

### standard
- yellow = claim/takeaway
- blue = evidence/method
- pink = limitation/risk
- green = orientation/definition
- orange = reusable quote

### minimal
- yellow = must-read
- pink = caution/limitation

### research
- yellow = claim
- blue = method
- pink = limitation
- green = concept/definition
- orange = quotable line
- underline-style note content should emphasize citation usefulness and scope boundaries

## Note templates

### Function note
`[主张] 这是作者要你接受的核心判断。`

### Plain-language note
`[人话] 把它翻成大白话：……`

### Reading instruction note
`[先看] 先抓结论，不要先陷进技术细节。`

### Critical note
`[限制] 这句决定了结论能推广到哪里，别省略。`

### Reuse note
`[复述] 以后给别人解释这篇论文时，可以直接借这个意思。`
